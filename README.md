사용 기술 및 라이브러리

이 프로젝트는 자바 기반 웹 개발 기술을 활용하고 있습니다. 주요 스택은 아래와 같습니다:

Java (JDK 8 이상) – 비즈니스 로직 구현에 사용되었습니다. JDBC를 통해 DB 연동, 서블릿/JSP를 통한 웹 처리 등이 모두 Java 코드로 작성되어 있습니다. Java 8 이상의 기능으로 try-with-resources 문법 등을 사용하였으며, Dockerfile 기준으로 JDK 21까지 호환 가능합니다.

JSP/Servlet – 웹 화면과 컨트롤러 역할을 합니다. 별도의 프레임워크 없이 JSP 파일 안에서 Java 코드로 DAO를 호출하고 결과를 표현하는 방식입니다. 서블릿 클래스는 별도로 존재하지 않고, JSP가 곧 서블릿으로 컴파일되어 동작합니다. (예: writePro.jsp가 폼 데이터를 처리하는 컨트롤러 역할을 합니다.) JSP 내에서 <%@ include %> 지시어로 공통 레이아웃(header.jsp, footer.jsp)을 포함시키는 등 서블릿/JSP 표준 기술을 활용했습니다.

Oracle Database & JDBC – Oracle 11g 혹은 12c 데이터베이스를 가정하고 있습니다. oracle.jdbc.driver.OracleDriver를 통해 연결하며, SQL 문장은 Oracle DB의 문법에 맞춰 작성되었습니다. 예를 들어 시퀀스 article_seq.nextval 사용, ROWNUM을 이용한 페이징, NVL 함수 등 Oracle 고유 기능을 사용했습니다. JDBC로 연결 정보(URL, 계정)를 직접 넣는 구조이며, 간단한 프로젝트이므로 Connection Pool 없이 매 요청 시 새 연결을 열고 닫는 방식을 취했습니다.

Apache Tomcat 9 – 웹 어플리케이션 서버로 사용됩니다. Dockerfile에서 tomcat:9.0-jdk21 이미지를 사용하고 있어, 빌드된 WAR 파일을 Tomcat에 배포하여 실행하게 됩니다. 따라서 서버 환경은 Tomcat 9 기반이고 JSP/서블릿 컨테이너로 동작합니다.

Maven – 프로젝트 의존성 관리 및 빌드에 사용됩니다. pom.xml에는 Oracle JDBC 드라이버나 O’Reilly 업로드 라이브러리 등이 추가되었을 가능성이 있지만, 만약 포함되지 않았다면 수동으로 lib에 jar를 넣어두었을 것입니다. Dockerfile에서 Maven을 통해 mvn package 명령을 실행하므로, Maven이 소스 코드를 컴파일하고 WAR 파일을 생성해줍니다.

README.md (프로젝트 설명서)
m4-news-project

m4-news-project는 뉴스 기사 공유 웹사이트입니다. 사용자는 뉴스를 읽고 댓글을 달 수 있을 뿐만 아니라, 직접 뉴스를 작성하여 게시할 수 있습니다. 또한 질문/건의 게시판(Q&A)을 통해 사용자 간 소통이 가능합니다. 본 프로젝트는 Java JSP/Servlet 기반으로 구축되었으며, Oracle 데이터베이스를 사용합니다.

주요 기능

회원 관리: 회원 가입, 로그인/로그아웃, 회원 정보 수정, 회원 탈퇴 기능을 제공합니다. 회원가입 시 선택한 직업에 따라 관리자의 승인이 필요한 계정(예: 기자 신청)과 일반 계정으로 구분됩니다.

뉴스 기사: 로그인한 사용자는 뉴스를 작성할 수 있습니다. 기사에는 **제목, 내용, 카테고리(정치/경제/사회 등)**를 지정할 수 있고, 이미지를 첨부할 수 있습니다. 작성된 기사는 메인 페이지 및 해당 카테고리 페이지에 노출됩니다.

이미지 업로드: 기사 작성 시 이미지를 한 개 이상 업로드 가능하며, 업로드된 이미지는 서버의 /upload 폴더에 저장되고 본문에 첨부됩니다.

뉴스 열람 및 반응: 방문자는 게시된 뉴스를 카테고리별로 모아보거나, 검색 기능을 통해 특정 키워드로 기사를 찾을 수 있습니다. 각 기사는 조회 수가 기록되며, 로그인 사용자는 기사에 공감(👍) 또는 비공감(👎) 반응을 남길 수 있습니다. 기사 하단에는 누적 공감/비공감 수치가 표시됩니다.

댓글 및 대댓글: 사용자는 기사에 댓글을 달 수 있습니다. 댓글에는 답글을 달 수 있어 여러 단계의 토론이 가능합니다. 댓글 작성자 또는 관리자만 자신의 댓글을 수정/삭제할 수 있습니다. 댓글에 대해서도 추천/비추천을 할 수 있으며, 추천 많은 댓글은 상위에 표시됩니다.

Q&A 게시판: 카테고리 중 하나인 Q&A를 통해 질문이나 건의사항을 올릴 수 있습니다. 다른 사용자가 답변 형태로 댓글을 달아 소통할 수 있습니다. (운영자는 이를 통해 피드백을 받을 수 있습니다.)

관리자 기능:

회원 승인: 기자로 가입 신청한 회원(APPROVAL_STATUS = 'W')을 목록으로 확인하고 승인('Y')하거나 거절할 수 있습니다.

게시글 블라인드: 부적절한 게시글이 신고되거나 할 경우 관리자가 해당 글을 블라인드 처리하여 일반 사용자에게 보이지 않도록 할 수 있습니다. (블라인드된 글은 관리자와 작성자에게만 보입니다.)

회원 차단: 문제를 일으키는 회원의 계정을 차단하여 로그인을 못 하도록 할 수 있습니다.

데이터베이스 구성

프로젝트가 사용되는 DB 스키마의 주요 테이블은 다음과 같습니다:

MEMBER 테이블: 회원 정보 저장

컬럼 예시: ID (PK), PW, NAME, BIRTH, GENDER, PHONE1, PHONE2, JOB, ZIP, ADDR1, ADDR2, REG_DATE(가입일, 기본값 SYSDATE), APPROVAL_STATUS('Y'=승인됨, 'W'=대기중), COMPANY(소속 회사명), STATUS(회원상태, 'blocked' 등), PROFILE_IMG(프로필 사진 파일명).

ARTICLE 테이블: 뉴스 게시글 저장

컬럼 예시: NUM (PK, 시퀀스 사용), WRITER(작성자 ID, MEMBER.ID 참조), SUBJECT, CONTENT, PASSWD(글 수정/삭제 비밀번호), WRITE_DATE, UPDATE_DATE, READCOUNT, LIKE_COUNT/GOOD_COUNT/SAD_COUNT/... (여러 감정 리액션 수, 현재는 미사용), AGREE_COUNT(공감 수), DISAGREE_COUNT(비공감 수), CATEGORI(카테고리 코드, 정수), EMPHASIZED(편집장 추천 여부, Boolean).

ARTICLE_FILES 테이블: 기사 첨부파일(이미지) 정보

컬럼: FILE_NUM (PK), ARTICLE_NUM(ARTICLE.NUM 참조), FILE_NAME (서버에 저장된 파일명).

COMMENTS 테이블: 댓글 저장 (계층 구조)

컬럼: COMMENT_ID (PK, 시퀀스), ARTICLE_ID(ARTICLE.NUM 참조), USER_ID(댓글 작성자 ID, MEMBER.ID 참조), CONTENT, COMMENT_DATE, REF(댓글 스레드 ID), RE_STEP(스레드 내 순서), RE_LEVEL(댓글 깊이, 0=원댓글).

COMMENT_EMOTION 테이블: 댓글 추천/비추천 (유저별 기록)

컬럼: ID(PK, 시퀀스), COMMENT_ID(COMMENTS.COMMENT_ID 참조), USER_ID(MEMBER.ID 참조), EMOTION_TYPE('like' 또는 'dislike'), CREATED_AT.

ARTICLE_EMOTION 테이블: 기사 공감/비공감 (유저별 기록)

컬럼: USER_ID, ARTICLE_NUM, EMOTION ('agree' 또는 'disagree'). (복합 PK로 사용자당 한 번만 공감 or 비공감 가능)

주의: Oracle DB를 미리 구축해야 합니다. 위 테이블 및 필요한 시퀀스, 제약조건 등을 생성해야 하며, src/main/resources에 DDL 스크립트가 제공되지 않았으므로 README의 정보를 참고하여 수동으로 설정해야 합니다. 기본적으로 team01 계정 사용자에게 테이블이 있고, 비밀번호 1234로 가정합니다. 개발/테스트 환경에서는 SQLite나 H2 등으로 변경도 가능하지만, 본 프로젝트는 Oracle에 최적화되어 있습니다.

실행 방법

개발 환경 설정: Java JDK와 Apache Tomcat이 설치되어 있어야 합니다. Maven도 설치되어 있으면 빌드에 도움이 됩니다. Oracle 데이터베이스가 실행 중이고 jdbc:oracle:thin:@58.73.200.225:1521:orcl로 접속 가능한 상태여야 합니다 (또는 소스 코드의 DB 연결 URL을 자신의 환경에 맞게 수정해야 합니다).

만약 Oracle DB를 직접 사용할 수 없다면, 데이터베이스 부분을 H2 등으로 교체하거나 Docker 이미지로 Oracle XE를 띄울 수 있습니다. 초보 사용자는 DB 설정이 가장 어려울 수 있으므로, 필요 시 선배 개발자나 자료를 참고하여 Oracle 11g Express 등을 설치해보세요.

DB 접속 계정(team01/1234)과 호스트(IP, 포트)은 UserDAO.java 등 코드 상단에 하드코딩되어 있습니다. 실제 환경에 맞게 수정하려면, 검색으로 "jdbc:oracle:thin:@" 문자열을 찾아 자신의 DB 정보로 변경해야 합니다. (향후 개선으로 properties 파일로 관리하는 것을 권장합니다.)

빌드 및 배포: Maven이 설치되어 있다면, 프로젝트 루트 경로에서 터미널을 열고 mvn package를 실행하십시오. Maven은 pom.xml에 따라 소스를 컴파일하고 WAR 파일(target/m4-news-1.0-SNAPSHOT.war)을 생성합니다.

Maven 빌드 중 테스트를 건너뛰려면 mvn package -DskipTests 옵션을 사용할 수 있습니다. (현재 프로젝트에는 JUnit 테스트가 없으므로 상관없습니다.)

WAR 파일이 생성되면, 이 파일을 Tomcat 서버의 webapps 폴더에 m4-news.war 등의 이름으로 복사한 후 Tomcat을 시작하면 자동으로 전개(deploy)됩니다. 또는 Tomcat의 관리 콘솔에서 WAR 배포를 할 수도 있습니다.

Docker로 실행 (선택사항): 이 프로젝트에는 Docker 지원이 포함되어 있습니다. 만약 Docker가 설치되어 있다면, 루트 폴더에 있는 Dockerfile을 사용하여 이미지를 빌드하고 컨테이너를 실행할 수 있습니다.

터미널에 docker build -t m4news . 명령을 실행하여 이미지를 빌드합니다. Maven이 컨테이너 내부에서 실행되어 WAR를 만들고, Tomcat에 배포하는 과정을 자동으로 처리합니다.

빌드가 완료되면 docker run -p 8080:8080 m4news로 컨테이너를 실행합니다. 그러면 로컬 8080 포트로 Tomcat이 구동되고, 브라우저에서 http://localhost:8080/으로 접속하면 애플리케이션을 볼 수 있습니다.

주의: Docker로 실행할 경우에도 Oracle DB에 접속해야 하므로, Docker 컨테이너에서 58.73.200.225 주소로 Oracle에 접근 가능해야 합니다. 사설망 환경이라면 해당 IP를 변경하거나 Oracle 컨테이너를 함께 띄워 네트워크를 연결해야 합니다.

초기 데이터 세팅: 애플리케이션이 뜨면, 처음에는 회원가입을 통해 몇 가지 계정을 만들어야 합니다. 특히 관리자용 계정을 하나 만들어두는 것이 좋습니다. 별도의 관리자 권한 필드는 없지만, 코드상에서 **아이디가 "admin"**인 사용자를 관리자로 간주하는 부분이 있으니, "admin" 아이디로 회원가입하고 (직업을 기자로 선택하여 즉시 승인되도록 하거나, DB에서 APPROVAL_STATUS를 'Y'로 변경) 그 계정으로 로그인하면 관리자 기능(회원 승인, 블라인드 등)을 사용할 수 있습니다.

테스트를 위해 몇 개의 기사 데이터를 등록해보세요. 카테고리를 골고루 지정하고 이미지를 첨부하면 메인 화면에 변화가 생깁니다. (예: 정치 뉴스 1개, 연예 뉴스 1개 작성하면 메인에서 최근 글로 보여집니다.)

댓글도 몇 개 달아보고, 다른 계정으로 로그인하여 답글, 추천/비추천 등을 시도해보세요. 이를 통해 각 기능이 제대로 동작하는지 확인할 수 있습니다.

애플리케이션 사용:

메인 페이지: 상단 내비게이션 바에서 카테고리를 선택하거나, 검색창에 키워드를 입력해 기사를 탐색할 수 있습니다. 메인에는 오늘자 최신 뉴스나 추천 많은 뉴스가 요약되어 보여집니다.

기사 쓰기: "글쓰기" 버튼을 클릭하면 기사 작성 페이지로 이동합니다. 제목, 내용 등을 입력하고 이미지를 선택한 후 등록합니다. 작성 성공 시 해당 글의 상세보기 페이지로 이동하며, "게시글이 성공적으로 등록되었습니다." 알림이 뜹니다.

기사 읽기 & 공감: 기사 상세보기 페이지에서는 제목, 작성자, 날짜, 본문, 첨부 이미지가 표시됩니다. 로그인한 사용자는 하단의 공감(👍) 또는 비공감(👎) 버튼을 눌러 자신의 의견을 표시할 수 있습니다. 한 번 누르면 취소하거나 변경은 불가하며, 버튼이 비활성(disabled)되어 중복 입력을 막습니다.

댓글 달기: 기사 하단에는 댓글 목록이 나옵니다. 로그인 상태에서 댓글 입력란에 내용을 쓰고 "등록" 버튼을 누르면 댓글이 추가됩니다. 댓글에는 작성 시간과 작성자명이 표시됩니다. 자신의 댓글에는 수정/삭제 버튼이 나타나며, 이를 통해 내용을 편집하거나 삭제할 수 있습니다.

대댓글 달기: 각 댓글 옆에는 "답글" 버튼이 있습니다. 눌러서 나타나는 작은 입력창에 내용을 쓰고 등록하면 해당 댓글의 아래에 들여쓴 형태로 답글이 추가됩니다. 여러 단계의 답글도 가능하며, 단계가 깊어질수록 더 들여쓰여 표시됩니다.

댓글 추천: 다른 사용자의 댓글에 대해 "👍 추천" 또는 "👎 비추천" 버튼을 클릭해 평가를 남길 수 있습니다. 추천 수와 비추천 수가 버튼 옆에 실시간으로 증가하며, 한 사용자가 같은 댓글에 중복해서 누를 수 없습니다.

프로필 이미지 변경: 상단 메뉴의 "마이페이지"에서 프로필을 수정할 수 있습니다. 이미지를 업로드하면 서버의 /upload 폴더에 저장되고, 다음 로그인 시 해당 이미지가 프로필로 표시됩니다. (기본 이미지는 default.png로 설정되어 있습니다.)

관리자 기능: 아이디가 "admin"인 계정으로 로그인하면, 상단에 "관리자 메뉴"가 나타납니다. 여기서 승인 대기 회원 목록을 보고 승인/거절을 처리하거나, 전체 회원 목록에서 특정 회원을 차단할 수 있습니다. 또한 각 기사 페이지에서 "블라인드" 버튼을 통해 해당 글을 숨길 수 있습니다. 블라인드된 글은 목록에서 사라지고, 상세보기도 관리자만 접근 가능합니다.

예시 및 화면 설명

회원가입/로그인: 첫 화면에서 "회원가입"을 눌러 계정을 생성합니다. 필수 정보를 입력하고 가입하면 자동 로그인되거나 로그인 페이지로 이동합니다. 로그인 시 잘못된 비밀번호를 입력하면 경고 메시지가 표시됩니다.

메인 페이지: 뉴스 사이트 메인에는 "오늘의 뉴스" 섹션이 있으며, 최신 기사나 추천 Top 기사 등이 카드 형식으로 나열됩니다. 각 카드에는 기사 이미지, 제목, 간략한 정보가 있고 클릭하면 상세 페이지로 이동합니다. (만약 등록된 뉴스가 없으면 "표시할 뉴스가 없습니다."라는 문구가 나옵니다.)

뉴스 목록: 상단 메뉴에서 "정치", "경제" 등의 카테고리를 클릭하면 해당 카테고리의 기사 목록이 표로 나타납니다. 번호, 제목, 작성자, 등록일, 조회수 열이 있으며, 제목을 클릭하면 상세보기로 갑니다. 목록 상단에는 카테고리 이름이나 검색어에 따른 현재 목록의 제목이 표시됩니다. 예를 들어 경제 뉴스를 보고 있다면 "경제 뉴스"가 제목으로 나타납니다. 우측 상단에는 "글쓰기" 버튼이 있어 새 글을 올릴 수 있습니다.

페이지네이션: 기사나 댓글이 많을 경우 페이지 넘김이 생깁니다. 하단에 [이전] [1][2]...[다음] 식으로 네비게이션이 표시되고, 현재 페이지는 굵게 강조됩니다. 한 페이지당 10개의 글이 기본이며, 이는 ArticleDAO의 pageSize로 설정되어 있습니다.

기사 상세 & 댓글: 기사 본문 아래에 댓글 목록이 최신순으로 나열됩니다. 댓글이 없는 경우 "등록된 댓글이 없습니다."라고 표시됩니다. 댓글이 있으면 각각 회색 박스로 구분되어 표시되고, 작성자 이름과 작성 시각, 내용이 나옵니다. 자신의 댓글에는 "(수정) (삭제)" 링크가 나타나고, 남의 댓글에는 추천/비추천 버튼이 보입니다. 답글은 들여쓰기되어 원댓글과 구분되며, 대화 형태로 나타납니다.

관리자 페이지: 관리자 메뉴에서 "승인 대기 회원"을 클릭하면, 아직 승인되지 않은 회원들의 리스트가 표시됩니다. 각 행에는 회원 ID, 이름, 가입일 등이 보이고 옆에 "✔ 승인" / "✖ 거절" 버튼이 있습니다. 승인을 누르면 해당 회원이 바로 활동할 수 있게 되고 리스트에서 제거됩니다. 거절을 누르면 회원이 삭제되며 리스트에서 제거됩니다. 처리 후에는 결과를 alert로 보여주고 관리자 페이지를 갱신합니다.

오류 처리: 예를 들어, 로그인 없이 글쓰기 페이지에 접근하면 session 체크를 통해 로그인 페이지로 리디렉션됩니다. 잘못된 접근이나 오류 상황 (DB 연결 실패 등)이 발생하면 콘솔에 오류가 출력되며, 사용자에게는 기본적으로 history.back() 또는 사용자 친화적인 메시지가 JSP에서 제공됩니다. (예: "잘못된 접근입니다." 경고 후 이전 페이지로 돌아감)

참고 및 향후 개선

환경 설정 변경: DB 접속 정보가 하드코딩되어 있으므로, 배포 시에는 보안을 위해 별도 설정파일로 분리하거나 환경변수로 관리하는 것이 좋습니다. 초보 단계에서는 .java 파일에서 URL과 계정을 수정하는 것으로 충분합니다.

프레임워크 도입: 본 프로젝트는 JSP와 DAO만으로 구성되어 구조적으로는 단순하지만, 규모가 커질 경우 유지보수가 힘들 수 있습니다. 추후 Spring MVC 같은 프레임워크를 학습하고 도입하면 MVC 구조를 견고히 하고 보안(예: SQL 인젝션 방지)이나 기능 구현을 편리하게 할 수 있습니다.

유효성 검사: 클라이언트 측(jQuery 또는 HTML5 Validation)과 서버 측에서 폼 입력값에 대한 검사가 필요합니다. 현재 비밀번호 최소 길이, 전화번호 숫자 체크 등 일부 자바스크립트로 처리되고 있으나, 더욱 강화할 수 있습니다.

테스트 데이터: 테스트를 위해 몇 개의 더미 데이터를 추가해 두었습니다. (ex: admin/1234 계정, 기본 뉴스 기사 3개 등) 실 서비스 시에는 초기 데이터 없이 운영하게 되므로, 운영자 계정을 먼저 만들고 필요한 설정을 해야 합니다.

학습 거리: 이 프로젝트를 통해 JSP/서블릿, JDBC 사용법, 그리고 기본적인 웹 개념(세션, 쿠키 등)을 익힐 수 있습니다. 코드에는 한글 주석이 풍부하게 포함되어 있으므로, 처음 웹 개발을 접하는 분도 흐름을 따라가기 수월할 것입니다. 궁금한 부분은 주석을 검색하거나 구글링하여 더 알아보세요.
